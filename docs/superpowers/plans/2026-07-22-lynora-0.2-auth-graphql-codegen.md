# Lynora 0.2 — Auth + GraphQL + Codegen Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add request auth helpers, GraphQL send/introspection, and code generation (curl / fetch / reqwest) to `lynora-core` and expose them in the desktop UI.

**Architecture:** Extend `RequestDocument` with optional `auth` + GraphQL fields (serde defaults keep 0.1 files valid). New modules: `auth`, `graphql`, `codegen`. Desktop gains auth picker, GraphQL mode, and Export snippet.

**Tech Stack:** Existing stack + `base64`, `sha2`, `hmac`, `percent-encoding` for auth; GraphQL over HTTP via reqwest (no full GraphQL client crate required for 0.2).

**Spec:** design §7 version 0.2.

---

### Task 1: Auth module
- Create `crates/lynora-core/src/auth.rs`
- Types: `None | Bearer | Basic | ApiKey | OAuth2Pkce | AwsSigV4`
- `apply_auth(headers, auth, vars) -> Result<Vec<(String,String)>>` merges auth into headers
- OAuth2 PKCE: `generate_pkce()` + `build_authorize_url` + `exchange_token` (HTTP)
- AWS SigV4: sign GET/POST with access key/secret/region/service (test with known fixture vectors or round-trip header presence)
- Tests for Bearer/Basic/ApiKey header injection + PKCE challenge shape

### Task 2: GraphQL module
- Create `crates/lynora-core/src/graphql.rs`
- `GraphQlRequest { url, query, variables_json, operation_name, headers }`
- `send` → POST JSON `{query, variables, operationName}` → `RestResponse`-compatible body
- `introspect(url, headers)` → standard introspection query
- Local HTTP server integration test

### Task 3: Codegen module
- Create `crates/lynora-core/src/codegen.rs`
- `Language::{Curl, Fetch, Reqwest}`
- `generate(prepared RestRequest) -> String`
- Unit tests snapshot expected snippets

### Task 4: Wire RequestDocument + prepare path
- Optional `auth`, `protocol` (`rest`|`graphql`), `graphql` body on documents
- `prepare_request` applies auth after var expansion
- Public re-exports in `lib.rs`

### Task 5: Desktop UI + Tauri commands
- Commands: `generate_code`, `send_graphql`, `introspect_graphql`, ensure `send_request` accepts auth
- UI: protocol toggle, auth type fields, GraphQL query/vars panes, Export dropdown

### Task 6: Verify + push
- `cargo test -p lynora-core`
- `cargo check -p lynora-desktop`
- Commit + push
