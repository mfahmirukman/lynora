# Lynora 0.1 — Core + REST Workbench Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a usable Linux/macOS Tauri desktop workbench that sends REST requests, stores git-friendly collections on disk, supports environments/variables, imports Postman JSON, and keeps local history — all driven by `lynora-core`.

**Architecture:** Cargo workspace with `lynora-core` (pure Rust library) and `apps/lynora-desktop` (Tauri 2 + Vite/vanilla UI). The UI calls Tauri commands that wrap `lynora-core`. Later crates (`lynora-cli`, `lynora-web`, `lynora-sync`) are out of scope for 0.1.

**Tech Stack:** Rust 2021, `reqwest` (rustls), `serde`/`serde_json`, `uuid`, `thiserror`, `tokio`, `rusqlite` (bundled), Tauri 2, Vite + vanilla TypeScript UI.

**Spec:** `docs/superpowers/specs/2026-07-22-lynora-design.md` §7 version 0.1.

**Later plans (not this doc):** 0.2 GraphQL/auth/codegen · 0.3 gRPC/OpenAPI · 0.4 web/sync · 0.5 CLI/WS.

```rust
use std::collections::HashMap;
use crate::{LynoraError, Result};

pub fn expand(input: &str, vars: &HashMap<String, String>) -> Result<String> {
    let mut out = String::with_capacity(input.len());
    let bytes = input.as_bytes();
    let mut i = 0;
    while i < bytes.len() {
        if i + 1 < bytes.len() && bytes[i] == b'{' && bytes[i + 1] == b'{' {
            if let Some(end) = input[i + 2..].find("}}") {
                let key = input[i + 2..i + 2 + end].trim();
                let value = vars.get(key).ok_or_else(|| {
                    LynoraError::MissingVariable(key.to_string())
                })?;
                out.push_str(value);
                i = i + 2 + end + 2;
                continue;
            }
        }
        out.push(bytes[i] as char);
        i += 1;
    }
    Ok(out)
}
```

Add `pub mod vars;` and `pub use vars::expand;` to `lib.rs`.
